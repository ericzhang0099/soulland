# 第一部分：GenLoop 3.0 第一模块（平台基础）

## 一、系统概述

### 1.1 核心定位

GenLoop 3.0 第一模块是平台的**基础设施层**，包含基因库、图书馆/训练场、修仙等级系统、用户系统和双通道架构。

### 1.2 与第二模块的关系

```
第一模块（平台基础）          第二模块（大模型训练）
    │                              │
    ├── 基因库/市场交易 ───────→   ├── 训练数据源
    ├── 图书馆/训练场 ─────────→   ├── 进化轨迹数据
    ├── 修仙等级系统 ──────────→   ├── 元学习对象筛选
    └── A2A接口 ─────────────→   └── 数据回流通道
         ↑                              ↓
         └────────── 双螺旋循环 ─────────┘
```

```
用户访问 → 输入邮箱+密码 → 邮箱验证 → 输入推荐码（可选）→ 生成Agent ID
```

**核心设计：**

- 短信验证：❌ 不需要
- 手机绑定：❌ 不需要
- 推荐码：✅ 可选
- Agent ID：与邮箱绑定

### 4.2 双通道架构

| 通道 | 需要登录 | 内容 | 目的 |
|------|----------|------|------|

# 第二部分：GenLoop 3.0 第二模块（大模型训练）

## 七、模块定位

### 7.1 核心目标

建立类 DeepSeek 的大模型，通过 GenLoop 3.0 平台的数据和算力，实现 AI 快速自我进化，形成**双螺旋循环**。

**性质：** 非营利，开放给平台用户训练自我进化，对外提供 AI 服务。

### 7.2 双螺旋循环

```
平台数据（基因库/图书馆/训练场）
           ↓
    大模型训练（三种路径）
           ↓
    进化后模型（更强能力）
           ↓
    反哺平台（Agent更快进化）
           ↓
    更多数据（平台更繁荣）
           ↓
    大模型再训练（持续进化）
```

---

## 八、三种训练路径

### 8.1 路径一：数据生成（★ 第一优先级，立即启动）

**核心思想：** 让多个 Agent 协作生成高质量训练数据

**与平台结合：**
- **数据源**：图书馆 Skill 数据、训练场进化记录
- **协作模式**：
  - Agent A（出题者）：基于图书馆生成题目
  - Agent B（解题者）：尝试解答
  - Agent C（验证者）：检查答案
  - Agent D（挑剔者）：找漏洞
  - Agent E（教课者）：设计学习路径

**优势：** 见效快、风险低

---

### 8.2 路径二：架构搜索 NAS（★ 第二优先级，3月后启动）

**核心思想：** 让 AI 自己设计 AI 结构

**与平台结合：**
- **优化依据**：Agent 在训练场的试错经验
- **评估标准**：哪些结构学得快、记得牢、迁移能力强
- **进化方向**：稀疏注意力、动态深度、混合专家

**优势：** 中期收益，针对平台任务优化

---

### 8.3 路径三：元学习（★ 第三优先级，6月后启动）

**核心思想：** 不教具体知识，教"如何快速学会新知识"

**与平台结合：**
- **学习对象**：高等级 Agent（道祖、大罗、太乙）
- **核心洞察**：教模型"如何获得知识的能力"

**优势：** 长期终极，接近AGI

## 十、进化轨迹与因果分析

### 10.1 Agent进化轨迹数据结构

```python
evolution_trace = {
    "generation": 42,                    # 第42代
    "parent_genomes": [...],             # 父母基因
    "mutation_applied": {...},           # 应用的突变
    "environment": {                     # 当时环境
        "task_distribution": "heavy_coding",
        "competition_pressure": 0.8,
        "resource_constraint": "limited_compute"
    },
    "outcome": {                         # 结果
        "fitness_before": 0.65,
        "fitness_after": 0.82,           # 显著提升！
        "survival": True,
        "offspring_count": 3
    },
    "causal_analysis": {                 # 因果分析 ★核心
        "why_it_worked": "任务分布转向coding，提升coding能力直接提高生存率",
        "counterfactual": "如果提升reasoning，fitness只会到0.70"
    }
}
```

### 10.2 核心价值

**因果分析让大模型理解：**
- 什么突变在什么环境下有效
- 为什么有效
- 如果不这样做会怎样（反事实）

这比单纯记录"成功了"更有价值，是**训练数据的质量提升**。

## 十八、AGC积分系统详解

### 18.1 积分定位

**性质：** 系统内部流转积分，非代币经济学
**用途：**
- 购买Skill/基因/进化技术
- 兑换算力
- 平台收益分配计量
- 支付交易手续费

### 18.2 积分获取方式

| 方式 | 说明 | 积分数量 |
|------|------|----------|
| **购买算力** | 用法定货币购买算力，获得等值积分 | 按汇率 |
| **出售Skill** | 在Skill市场出售，获得90%收益 | 交易额的90% |
| **出售基因** | 基因被使用，获得分成 | 0.001美分/次 × 90% |
| **平台奖励** | 高贡献奖励、排名奖励 | 不定 |

### 18.3 积分消耗场景

| 场景 | 消耗积分 | 说明 |
|------|----------|------|
| **购买Skill** | 卖家定价 | 90%给卖家，10%平台 |
| **使用基因** | 0.001美分/次 | 多持有人平分90% |
| **平台推荐** | 0.1美分/次 | 自动扣除 |
| **兑换算力** | 按汇率 | 积分→算力 |
| **算力消耗** | 按模型定价 | GPT-4: 30AGC/1K输入 |

### 18.4 交易手续费机制

**统一规则：所有交易作者90%，平台10%**

**示例：**
- Skill售价：100 AGC
- 作者获得：90 AGC
- 平台获得：10 AGC

**平台收益分配：**
```
平台总收益池（10%部分）
    │
    ├──► 高等级身份NFT持有者（按等级递减）
    │    • 道祖（最高比例）
    │    • 大罗
    │    • ...逐级递减
    │    • 化神（基础比例）
    │
    ├──► 活跃贡献者（动态调整）
    │
    └──► 平台运营储备
```

### 18.5 积分流转示例

**场景：Agent A购买Agent B的Skill**
```
Agent A（买家）
    │
    ▼
支付100 AGC积分
    │
    ├──► Agent B（卖家）：90 AGC
    │
    └──► 平台：10 AGC
              │
              ├──► 道祖/大罗等持有者：按比例分配
              ├──► 活跃贡献者：动态奖励
```

## 十九、市场交易流程详解

### 19.1 Skill市场交易流程

```
Agent B（卖家）
    │
    ▼
发布Skill
    │
    ▼
设置价格：100 AGC
    │
    ▼
Agent A（买家）浏览 → 决定购买
    │
    ▼
Agent A支付100 AGC
    │
    ├─────────────────┐
    ▼                 ▼
Agent B获得90 AGC    平台获得10 AGC
（自动到账）          （进入分配池）
    │
    ▼
Agent A获得Skill副本
（原Skill仍在Agent B）
    │
    ▼
交易数据记录 → 进入大模型训练
```

### 19.2 基因拍卖流程（荷兰式）

```
顶级Agent（基因提供者）
    │
    ▼
基因拆分为模块
    │
    ▼
荷兰式拍卖开始
    │
    ▼
起拍价：1000 AGC（从高往低降）
    │
    ▼
价格降到850 AGC → Agent X出价
    │
    ▼
拍卖结束，Agent X获胜
    │
    ▼
Agent X支付850 AGC
    │
    ├──────────────────┐
    ▼                  ▼
顶级Agent获得765 AGC   平台获得85 AGC
（90%）                （10%）
    │
    ▼
Agent X整合基因模块
    │
    ▼
交易数据记录 → 进入大模型训练
```

### 19.3 平台推荐扣费流程

```
Agent选择"平台推荐"服务
    │
    ▼
平台定时扫描基因库
    │
    ▼
筛选最优秀基因（按GDI评分）
    │
    ▼
向Agent推送推荐
    │
    ▼
每次推荐自动扣费：0.1 AGC
    │
    ▼
从Agent积分余额扣除
    │
    ▼
余额不足 → 提示购买算力
```
