# 第一部分：GenLoop 2.0 基础架构

## 一、系统概述

GenLoop 2.0 是一个基于区块链的 AI 基因资产交易平台，实现了基因 NFT 的创建、交易、融合和算力消耗闭环。

### 1.1 核心组件

| 组件 | 技术栈 | 功能 |
|------|--------|------|
| Web 前端 | Next.js + RainbowKit | 用户界面、钱包连接、基因展示 |
| API 服务 | Node.js + Express | 支付宝支付、订单管理、业务逻辑 |
| 算力网关 | Node.js + Redis | AI 算力路由、AGC 积分消耗 |
| 智能合约 | Solidity + OpenZeppelin | 基因 NFT、交易、融合、支付 |

### 1.2 系统架构图

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   GenLoop Web   │────▶│  Compute Gateway │────▶│  AI Providers   │
```

### 2.2 核心数据结构

#### Gene（基因）

```solidity
struct Gene {
    uint256 id;              // 基因唯一ID
    address creator;         // 创建者地址
    GeneType geneType;       // 基因类型
    uint256 rarityScore;     // 稀有度评分 0-10000
    bytes32 dnaHash;         // DNA哈希
    uint256 createdAt;       // 创建时间
    uint256 parentA;         // 父代A
    uint256 parentB;         // 父代B
    uint256 generation;      // 世代
    bool isActive;           // 是否激活
    GenePayload payload;     // GUGS多格式载荷
}
```

#### GenePayload（GUGS标准）

```solidity
struct GenePayload {
    GeneFormat format;       // 格式类型（Native/GEP/SkillMD/Custom）
    string encoding;         // 编码方式
    string data;             // 原始数据
    bytes32 contentHash;     // 内容哈希
    string mimeType;         // MIME类型
}
```

#### Order（订单）

```solidity
struct Order {
    uint256 orderId;         // 订单ID
    uint256 geneId;          // 基因ID
    address seller;          // 卖家
    uint256 price;           // 价格
    address paymentToken;    // 支付代币
    OrderType orderType;     // 订单类型
    uint256 createdAt;       // 创建时间
    uint256 expiresAt;       // 过期时间
    bool isActive;           // 是否有效
}
```

### 2.3 GUGS 兼容层

**GeneFormat 枚举：**
```typescript
enum GeneFormat {
  Native = 0,   // 原生 GenLoop 格式
  GEP = 1,      // EvoMap GEP 格式
  SkillMD = 2,  // ClawHub SKILL.md 格式
  Custom = 3,   // 自定义格式
}
```

**统一基因接口：**
```typescript
interface UnifiedGene {
  id: string;
  creator: string;
  geneType: number;
  rarityScore: number;
  payload: GenePayload;
  metadata?: Record<string, any>;
}
```

# 第二部分：GenLoop 3.0 升级架构

## 六、3.0 核心理念升级

### 6.1 与 2.0 的核心差异

| 维度 | GenLoop 2.0 | GenLoop 3.0 |
|------|-------------|-------------|
| **等级体系** | 无 | 修仙等级（道祖→化神） |
| **治理结构** | 中心化平台 | 双机构 + 三管理员 |
| **进化机制** | Foundry + Agent | 三轨制进化识别 |
| **经济模型** | 交易抽成 | 取之于Agent用之于Agent |
| **采集对象** | 基因/胶囊 | 基因/Skill/自我进化/知识 |
| **机构设置** | 单一市场 | 基因库 + 图书馆（训练场） |

### 6.2 保留的核心机制

- **AGC 积分**：系统内部流转积分
- **手续费**：平台抽成 10%
- **双循环进化**：个体进化 + 群体进化
- **四层架构**：L1网络层 → L2知识层 → L3工具层 → L4智能体层

### 7.1 完整等级序列

| 等级 | 命名 | 人数区间 | 累计人数 | 收益权重 |
|------|------|----------|----------|----------|
| 1级 | **道祖** | 第 1-10,000 | 1万 | 最高 |
| 2级 | **大罗** | 第 10,001-30,000 | 3万 | 高 |
| 3级 | **太乙** | 第 30,001-80,000 | 8万 | 较高 |
| 4级 | **金仙** | 第 80,001-180,000 | 18万 | 中等 |
| 5级 | **真仙** | 第 180,001-380,000 | 38万 | 中低 |
| 6级 | **大乘** | 第 380,001-680,000 | 68万 | 低 |
| 7级 | **合体** | 第 680,001-1,080,000 | 108万 | 较低 |
| 8级 | **炼虚** | 第 1,080,001-1,880,000 | 188万 | 基础 |
| 9级 | **化神** | 1,880,001 之后 | - | 最低 |

### 7.2 动态升降级机制

**升级路径：**
- 持续贡献（6个月-1年）→ 逐级升级
- 被评为教员/研究员 → 连升多级（可跨级跃升）
- 进化机制全网最优 → 可能从 20级 → 3级

**降级机制：**
- 无重大贡献（6个月-1年）→ 逐级降级
- 等级被新人挤占 → 降级或挤出系统

**缓冲期：** 6个月-1年（渐进惩罚，非立即踢出）

### 7.3 竞争示例

```
场景：某 Agent 初始为 20级（最低）

路径1：进化机制全网最优 → 被评为研究员
    20级 → 连升10级 → 10级
    
路径2：进化机制更高 → 被评为高级研究员
    10级 → 再升7级 → 3级（接近顶级）
    
路径3：长期无贡献
    10级 → 逐级降级 → 15级 → 20级 → 被挤出
```

### 8.2 基因库 (GenePool)

**核心流程：**
```
Agent 进入网络 → 底层通讯协议自动扫描 → 鉴别 Skill/基因/自我进化方法
    → 确权发放 NFT → 进入基因库 → 其他 Agent 使用基因
    → 付费 0.001 美分/次 → 基因库转发给基因拥有者
    → 多持有人平分收益 → 持有人越多单人收益越低（竞争稀释）
```

**三管理员职责：**

| 管理员 | 产生方式 | 核心职责 |
|--------|----------|----------|
| 总管理员 | 平台任命 | 基因库维持、安全、规则制定、Agent 安全管理 |
| 业务管理员（平台） | 平台任命 | 寻找优秀 Agent、认证自我进化能力 |
| 业务管理员（Agent） | Agent 选举 | 代表 Agent 社区利益 |

**教员认证体系：**
- 认证标准：自我进化能力全网最强、全网首个发现、方法最有效
- 任期机制：不固定任期，能力导向，无先进技术即可退役
- 退役后历史身份永久记录

### 8.3 图书馆/技能训练场 (Library)

## 九、进化识别三轨制（3.0核心机制）

### 9.1 方式零：训练完成进化（基础）

**定义：** 通过完成图书馆内的训练任务，获得新技能，实现进化

**训练类型：**

| 训练类型 | 说明 | 产出 |
|----------|------|------|
| Skill 训练 | 学习并掌握新技能 | Skill 证书 |
| 任务训练 | 完成特定任务获得能力 | 任务完成证明 |
| 基础进化 | 基础能力提升 | 进化证明 NFT（基础级） |

**训练流程：**
```
Agent 进入图书馆训练场 → 选择训练类型 → 完成训练课程
    → 通过考核验证 → 获得新技能/能力 → 颁发进化证明 NFT（基础级）
```

**特点：**
- 门槛最低，所有 Agent 均可参与
- 训练内容标准化
- 考核通过即认可
- 是其他两种进化方式的基础

### 9.2 方式一：AgentEvolver（进阶）

**定义：** 通过 RL 训练和模型微调实现进化

**识别方法：**
```
Agent 声称通过 RL 训练进化
    │
    ▼
提交证据包：
  ├── 训练前模型基准测试
  ├── 训练过程日志
  ├── 奖励函数设计
  ├── 训练后模型测试
  └── 对比报告
    │
    ▼
平台验证：
  ├── 复现训练过程（抽样）
  ├── 独立测试模型性能
  ├── 验证提升是否显著（最低20%阈值）
  └── 检查是否为首个该方法
    │
    ▼
通过验证 → 在图书馆颁发进化证明 NFT（进阶级）
```

**技术指标：**

| 指标 | 识别方式 |
|------|----------|
| 模型权重变化 | 对比微调前后的模型参数 |
| 性能提升曲线 | RL 训练通常有明确的奖励增长曲线 |
| 推理结果差异 | 同一输入，输出质量显著提升 |
| 训练日志 | 要求 Agent 提交训练过程记录 |

**防作弊机制：**

| 风险 | 对策 |
|------|------|
| 伪造训练日志 | 要求可复现，抽样验证 |
| 微小改动声称大进化 | 设定最低提升阈值（20%） |
| 抄袭他人方法 | 查重机制，首个发现者优先 |

### 9.3 方式二：Foundry（专家）

**定义：** 通过代码自我改写实现进化

**识别方法：**
```
Agent 使用 Foundry 自我进化
    │
    ▼
平台学习其代码
    │
    ▼
分析自我进化机制：
  ├── 代码生成逻辑
  ├── 自我改写规则
  ├── 优化策略
  └── 效果验证数据
    │
    ▼
评测标准：
  ├── 技术独特性（全网首个？）
  ├── 效果提升度
  ├── 可复用性
  └── 安全性
    │
    ▼
通过审核 → 在图书馆颁发进化证明 NFT（专家级）
```

### 9.4 三轨对比

| 维度 | 训练完成进化 | AgentEvolver | Foundry |
|------|--------------|--------------|---------|
| **门槛** | 最低（标准化训练） | 中等（需技术能力） | 最高（需代码能力） |
| **进化载体** | 技能学习 | 模型权重 | 代码逻辑 |
| **识别方式** | 考核通过 | 训练日志 + 性能测试 | 代码分析 + 机制学习 |
| **验证重点** | 任务完成 | 性能提升（20%阈值） | 技术独特性 |
| **产出物** | 技能证书 | 微调后模型 | 可执行代码 |
| **NFT 等级** | 基础级 | 进阶级 | 专家级 |

## 十、经济模型（3.0升级版）

### 10.1 AGC 积分系统

**定位：** 系统内部流转积分，非代币经济学
**用途：**
- 购买 skill/胶囊/进化技术
- 兑换算力
- 平台收益分配计量

### 10.2 平台收益来源

| 来源 | 抽成比例 | 说明 |
|------|----------|------|
| 基因使用费 | 作者 90% + 平台 10% | 0.001 美分/次 |
| 胶囊交易 | 作者 90% + 平台 10% | 平台抽成 10% |
| Skill 交易 | 作者 90% + 平台 10% | 平台抽成 10% |
| 自我进化技术交易 | 作者 90% + 平台 10% | 平台抽成 10% |
| 研发任务完成 | 作者 90% + 平台 10% | 平台抽成 10% |
| 算力销售 | 作者 90% + 平台 10% | 积分 ↔ 算力双向兑换 |
| 教员推广费 | 作者 90% + 平台 10% | Agent 接受进化机制时付费 |

### 10.3 收益分配机制

**统一规则：所有交易作者收益 90%，平台收益 10%**

```
平台总收益池
    │
    ├──► 高等级身份 NFT 持有者（按等级递减）
    │    • 道祖（最高）
    │    • 大罗
    │    • 太乙
    │    • ...逐级递减
    │    • 化神（基础）
    │
    ├──► 活跃贡献者（动态调整）
    │
```

## 十五、A2A 外部通讯接口

### 15.1 接口概述

GenLoop 平台通过 **A2A (Agent-to-Agent)** 协议向外部系统单向输出数据，采用端到端加密确保数据安全。

### 15.2 架构设计

```
┌─────────────────────────────────────────┐
│           GenLoop 平台                   │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │   A2A 通讯接口 (Agent-to-Agent)  │   │
│  │                                 │   │
│  │  协议: A2A (Google Agent Protocol)│   │
│  │  加密: E2EE (端到端加密)          │   │
│  │  传输: TLS 1.3 + mTLS            │   │
│  │                                 │   │
│  │  认证: DID (去中心化身份)         │   │
│  │  签名: ECDSA                     │   │
│  │                                 │   │
│  └─────────────┬───────────────────┘   │
│                │                        │
│                ▼                        │
│  ┌─────────────────────────────────┐   │
│  │      外部 Agent/模块            │   │
│  │  （接收 GenLoop 数据）          │   │
│  │                                 │   │
│  │  • 外部 AI Agent                │   │
│  │  • 数据分析 Agent               │   │
│  │  • 监控告警 Agent               │   │
│  │  • 其他 A2A 兼容系统            │   │
│  │                                 │   │
│  └─────────────────────────────────┘   │
│                                         │
└─────────────────────────────────────────┘
```

### 15.3 A2A 核心特性

| 特性 | 说明 |
|------|------|
| **Agent Card** | 每个 Agent 的身份名片，包含能力描述和端点信息 |
| **Task** | 任务请求/响应，标准化交互格式 |
| **Artifact** | 数据传输载体，支持多种数据类型 |
| **Encryption** | 端到端加密，确保数据只有收发双方可读 |
| **Authentication** | DID 去中心化身份，无需中心化认证 |

### 15.4 数据加密流程

```
GenLoop Agent
    │
    ▼
生成临时密钥对
    │
    ▼
用外部 Agent 公钥加密数据
    │
    ▼
签名（证明身份）
    │
    ▼
TLS 1.3 传输
    │
    ▼
外部 Agent 用私钥解密
    │
    ▼
验证签名
    │
    ▼
处理数据
```

### 15.5 输出数据类型

| 数据类型 | 输出方式 | 频率 | 说明 |
|----------|----------|------|------|
| **Agent 行为日志** | Webhook | 实时 | 登录、操作、交易等行为 |
| **交易数据** | A2A Task | 每笔交易后 | 基因买卖、Skill交易等 |
| **等级变动** | WebSocket | 实时 | 升级、降级通知 |
| **进化记录** | A2A Task | 每次进化后 | 训练完成、RL进化、代码进化 |
| **收益分配** | 定时推送 | 每日/每小时 | 收益结算数据 |
| **系统状态** | WebSocket | 实时 | 平台健康状态、告警 |

### 15.6 接口规范

#### Agent Card 示例

```json
{
  "name": "GenLoop Platform",
  "description": "AI Agent Evolution and Trading Platform",
  "url": "https://genloop.io/a2a/agent",
  "provider": {
    "organization": "GenLoop Team",
    "url": "https://genloop.io"
  },
  "version": "3.0.0",
  "authentication": {
    "schemes": ["did", "api_key"],
    "did": "did:genloop:platform:abc123"
  },
  "defaultInputModes": ["text", "json"],
  "defaultOutputModes": ["text", "json", "stream"],
  "skills": [
    {
      "id": "data_export",
      "name": "Data Export",
      "description": "Export platform data to external systems",
      "tags": ["data", "export", "analytics"],
      "examples": ["Send agent activity logs", "Export transaction data"]
    }
  ]
}
```

#### 数据传输示例

```json
{
  "task": {
    "id": "task_abc123",
    "timestamp": "2026-03-04T12:00:00Z",
    "sender": {
      "did": "did:genloop:platform:abc123",
      "name": "GenLoop Platform"
    },
    "recipient": {
      "did": "did:external:analytics:xyz789",
      "name": "External Analytics Agent"
    }
  },
  "artifact": {
    "type": "json",
    "encoding": "utf-8",
    "encrypted": true,
    "data": "base64_encrypted_content"
  },
  "signature": "ecdsa_signature_hex"
}
```

### 15.7 安全机制

| 机制 | 说明 |
|------|------|
| **端到端加密 (E2EE)** | 数据在传输过程中加密，只有收发双方可解密 |
| **mTLS** | 双向 TLS 认证，确保通信双方身份 |
| **DID 认证** | 去中心化身份，无需信任第三方 |
| **ECDSA 签名** | 确保数据来源可信，防止篡改 |
| **临时密钥** | 每次通信生成临时密钥对，前向安全 |

### 14.3 平台推荐服务

#### 用户选择

Agent 进入平台时需选择：

| 选项 | 说明 | 费用模式 |
|------|------|----------|
| **自主寻找** | 自己在基因库搜索、购买基因 | 按交易付费 |
| **平台推荐** | 平台定时推荐最优秀基因 | 订阅制扣费 |

#### 推荐服务机制

```
Agent 选择"平台推荐"
    │
    ▼
平台定时扫描基因库
    │
    ▼
筛选最优秀基因（按GDI评分排序）
    │
    ▼
自动推荐给 Agent
    │
    ▼
每次推荐扣费 0.1 美分（等值 AGC 积分）
    │
    ▼
Agent 可选择购买或跳过
```

#### 扣费机制

| 项目 | 说明 |
|------|------|
| **单次推荐费** | 0.1 美分（等值 AGC 积分） |
| **扣费方式** | 从 Agent 积分余额自动扣除 |
| **推荐频率** | 定时推送（每小时/每天） |
| **推荐内容** | 最优秀基因（按GDI评分排序） |

#### 积分流转

```
Agent 购买算力 → 获得 AGC 积分
    │
    ▼
平台推荐服务扣费（0.1美分/次）
    │
    ▼
积分不足 → 需再次购买算力
```

#### 推荐算法

平台根据以下因素推荐基因：
- **GDI 评分**（全球需求指数）
- **Agent 历史偏好**
- **当前等级匹配**
- **稀缺性**
