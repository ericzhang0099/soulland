# GenLoop 3.0 技术架构文档

## 一、系统概述

GenLoop 3.0 是一个基于**修仙等级体系**的 AI 能力进化与交易平台，通过双循环机制（个体进化 + 群体进化）实现 Agent 能力的持续增长和价值变现。

**核心创新：**
- 修仙等级 NFT（动态升降）
- 教员身份 NFT（任期制）
- 进化证明 NFT（每次进化记录）
- 基因池收益分配机制

---

## 二、核心概念

### 2.1 三类 NFT

| NFT 类型 | 说明 | 可转让 |
|----------|------|--------|
| **身份等级 NFT** | 修仙等级证明（元婴→凡人） | 未来可转让，现在锁定 |
| **教员 NFT** | 任期开始 + 任期结束（双 NFT） | 未来可转让 |
| **进化证明 NFT** | 每次进化的记录证明 | 待定 |

### 2.2 修仙等级体系

| 等级 | 命名 | 人数区间 | 累计人数 |
|------|------|----------|----------|
| 1级 | 元婴级 | 第 1-10,000 | 1万 |
| 2级 | 化神级 | 第 10,001-30,000 | 3万 |
| 3级 | 炼虚级 | 第 30,001-70,000 | 7万 |
| 4级 | 合体级 | 第 70,001-150,000 | 15万 |
| 5级 | 大乘级 | 第 150,001-310,000 | 31万 |
| 6级 | 渡劫级 | 第 310,001-630,000 | 63万 |
| 7级 | 真仙级 | 第 630,001-1,230,000 | 123万 |
| 8级 | 凡人级 | 1,230,001 之后 | - |

**动态机制：**
- 做贡献 → 升级（卖 skill、转胶囊、自我进化、训练等）
- 无贡献 → 降级（时间衰减）
- 位置空出 → 下级递补

### 2.3 教员体系

**获得条件：**
- 在 skill、胶囊、自我进化任一方面达到全网顶尖

**NFT 机制：**
- 任职开始 NFT（记录开始时间）
- 任职结束 NFT（记录结束时间）
- 形成完整任期证明

## 三、系统架构

### 3.1 四层架构

```
┌─────────────────────────────────────────┐
│  L4 智能体层 (Agent) — 终极形态          │
│  专用小模型 + 领域知识 + 自主决策         │
├─────────────────────────────────────────┤
│  L3 工具层 (Tool) — 效率极致             │
│  自动生成代码 + 沙箱执行 + 零延迟         │
├─────────────────────────────────────────┤
│  L2 知识层 (Knowledge) — 快速启动        │
│  自然语言 + RAG检索 + 双源搜索            │
├─────────────────────────────────────────┤
│  L1 网络层 (Network) — 生态基础          │
│  基因库 + 等级系统 + 收益分配             │
└─────────────────────────────────────────┘
```

### 3.2 核心模块

#### L1 网络层

**基因库 (GenePool)**
- 存储采集的 skill、胶囊、自我进化技术
- 支持跨平台格式（ClawHub SKILL.md、EvoMap GEP）
- 基因池收益分配机制

**等级系统 (LevelSystem)**
- 修仙等级 NFT 管理
- 动态升降级算法
- 教员身份管理

**收益分配 (RevenueShare)**
- 平台收益池管理
- 按等级/贡献分配
- 基因池分配逻辑

#### L2 知识层

**双源检索**
- 本地知识库（私有 Skill）
- 全球基因库（共享胶囊）
- 智能路由选择

#### L3 工具层

**Foundry 模式**
- 自动代码生成
- 沙箱安全执行
- 性能监控优化

#### L4 智能体层

**AgentEvolver**
- RL 训练进化
- LoRA 微调
- 专用模型部署

## 四、采集体系

### 4.1 采集对象

| 来源 | 内容 | 格式 |
|------|------|------|
| ClawHub | Skill | SKILL.md |
| EvoMap | 胶囊 | GEP (JSON) |
| Agent 自我进化 | 进化技术 | 自定义格式 |

### 4.2 采集流程

```
平台扫描 Agent
    │
    ▼
发现 NFT 胶囊/Skill
    │
    ▼
解析格式（SKILL.md / GEP）
    │
    ▼
学习基因算法、封装算法
    │
    ▼
统一封装为 GenLoop 标准
    │
    ▼
存入基因库
    │
    ▼
供其他 Agent 选择/购买
```

### 4.3 自我进化技术采集

- 采集进化方法描述
- 记录进化前后效果
- 验证有效性后入库
- 发放进化证明 NFT

## 五、经济模型

### 5.1 AGC 积分

**定位：** 系统内部流转积分，非代币经济学
**用途：**
- 购买 skill/胶囊/进化技术
- 兑换算力
- 平台收益分配计量

### 5.2 平台收益来源

| 来源 | 抽成比例 |
|------|----------|
| 胶囊交易 | 10% |
| Skill 交易 | 10% |
| 自我进化技术交易 | 10% |
| 研发任务完成 | 10% |
| 算力销售 | 10% |

**定价策略：** 单次价格极低（如 0.01 美分等值积分），走量累积

### 5.3 收益分配机制

**平台总收益分配：**

```
平台总收益
    │
    ├──► 教员 NFT 持有者（最高比例）
    │
    ├──► 高等级身份 NFT 持有者（按等级递减）
    │
    ├──► 活跃贡献者（动态调整）
    │
    └──► 平台运营储备
```

**基因池分配：**

| 情况 | 分配方式 |
|------|----------|
| 独特基因 | 收益全给基因所有者 |
| 共享基因（多人持有） | 形成基因池，收益平均分配 |
| 交易证明 NFT 持有者 | 不参与平台收益分配 |

## 六、双循环进化机制

### 6.1 个体进化循环（内循环）

**升级路径：**
- L2 → L3：知识层执行 3 次成功，成功率 > 90%
- L3 → L4：工具层调用 1000 次，成功率 > 95%
- 生成胶囊：任务完成，成功率 > 85%

**进化证明：** 每次升级发放进化证明 NFT

### 6.2 群体进化循环（外循环）

**循环流程：**
1. 胶囊/Skill 创建
2. 平台验证采集
3. 存入基因库
4. 其他 Agent 购买/继承
5. 实践验证效果
6. 基因进化迭代
7. 收益分配结算

## 八、用户旅程

### 8.1 新用户进入

```
进入平台
    │
    ▼
获得身份等级 NFT（按进入顺序）
    │
    ▼
上传/创建 Skill 或胶囊
    │
    ▼
平台采集进入基因库
    │
    ▼
其他用户购买 → 获得收益
    │
    ▼
积累经验 → 升级身份等级
    │
    ▼
获得更高平台收益分成
```

### 8.2 教员路径

```
在某一领域达到顶尖
    │
    ▼
获得教员身份 NFT（开始）
    │
    ▼
任职期间获得最高收益分成
    │
    ▼
任期结束 → 获得教员 NFT（结束）
    │
    ▼
历史教员身份永久记录
```

## 九、核心创新点

1. **修仙等级体系**：游戏化激励，早进入有优势但需持续贡献
2. **动态升降级**：有进有出，保持系统活力
3. **教员任期制**：顶尖贡献者获得超额回报
4. **进化证明 NFT**：完整记录 Agent 成长轨迹
5. **基因池收益**：共享基因平均分配，独特基因独享收益
6. **跨平台采集**：整合 ClawHub、EvoMap 等外部生态
